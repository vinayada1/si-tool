# si-generator

A CLI tool that generates `security-insights.yml` files for GitHub repositories by scanning local files and the GitHub API.

## Features

- 🔍 **Local Scanning**: Automatically detects security policies, code of conduct, licenses, and CI/CD workflows
- 🔌 **GitHub API Integration**: Fetches branch protection rules, MFA status, and repository metadata
- 🎯 **Interactive Mode**: Prompts for missing fields when needed
- ✅ **Validation**: Validates generated YAML against the official CUE schema
- 📝 **Dry Run**: Preview output before writing to file

## Installation

### From Source

```bash
go install github.com/si-generator/cmd/si-gen@latest
```

### Build Locally

```bash
git clone https://github.com/your-org/si-generator.git
cd si-generator
go build -o si-gen ./cmd/si-gen
```

## Usage

### Generate a security-insights.yml

```bash
# Generate with defaults
si-gen generate

# Specify output path
si-gen generate --output .github/security-insights.yml

# Interactive mode (prompts for missing fields)
si-gen generate --interactive

# Use GitHub API for additional data
si-gen generate --token $GITHUB_TOKEN

# Preview output without writing
si-gen generate --dry-run

# Force overwrite existing file
si-gen generate --force

# Verbose output
si-gen generate --verbose
```

### Validate a security-insights.yml

```bash
# Validate default path
si-gen validate

# Validate specific file
si-gen validate --input .github/security-insights.yml

# Use custom schema
si-gen validate --input security-insights.yml --schema custom-schema.cue
```

### Check Version

```bash
si-gen version
```

## CLI Flags

### Generate Command

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--output` | `-o` | Output file path | `./security-insights.yml` |
| `--dry-run` | | Print to stdout | `false` |
| `--interactive` | `-i` | Prompt for missing fields | `false` |
| `--token` | `-t` | GitHub token | `$GITHUB_TOKEN` |
| `--repo-url` | `-r` | Override repository URL | auto-detected |
| `--force` | `-f` | Overwrite existing file | `false` |
| `--verbose` | `-v` | Enable debug logging | `false` |

### Validate Command

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--input` | `-i` | Input file to validate | `./security-insights.yml` |
| `--schema` | `-s` | Custom CUE schema file | built-in schema |
| `--verbose` | `-v` | Enable debug logging | `false` |

## What Gets Detected

### Local Repository Scanning

- **Security Policy**: `.github/SECURITY.md`, `SECURITY.md`
- **Code of Conduct**: `.github/CODE_OF_CONDUCT.md`, `CODE_OF_CONDUCT.md`
- **License**: `LICENSE`, `LICENSE.md`, `LICENSE.txt`
- **Code Scanning Tools**: CodeQL, Snyk, SonarCloud, Semgrep, Trivy, and more
- **Dependency Tools**: Dependabot, Renovate
- **Fuzzing**: Go fuzz tests, OSS-Fuzz integration
- **Audit History**: `AUDIT.md`, `audits/` directory
- **Dependencies**: `package.json`, `go.mod`, `requirements.txt`, etc.

### GitHub API Data (requires token)

- Repository URL and metadata
- Branch protection rules
- Code review requirements
- MFA enforcement status (requires `admin:org` scope)
- License information
- Community health files
